### 版本差異對時間線的影響

**金庸小說版本演變：**
1. **連載版**（1955-1972）：最初在報刊連載的版本
2. **修訂版**（1970-1980）：金庸親自修訂的版本
3. **新修版**（1999-2006）：金庸晚年再次修訂的版本

**時間線差異：**
- 不同版本在情節細節、人物關係、時間設定上可能有差異
- Google Sheets 編年史資料來源標註為「新修版金庸編年史(多明戈)」

## Technical Trends and Innovation

### 互動地圖與時間軸視覺化技術

對於金庸作品歷史地圖專案，以下技術趨勢與創新模式具有重要參考價值：

#### 1. 地圖視覺化技術

**現代地圖庫：**
- **Leaflet：** 開源、輕量級、易於整合歷史地圖底圖
- **Mapbox：** 提供豐富的地圖樣式與自訂功能
- **OpenLayers：** 適合複雜的地理資料視覺化

**歷史地圖整合：**
- **Old Maps Online API：** 可整合歷史地圖底圖
- **歷史地圖資料庫：** 可取得不同時期的歷史地圖
- **地圖圖層系統：** 支援多圖層疊加（歷史地圖 + 現代標記）

**技術優勢：**
- 支援互動式標記與彈出視窗
- 可整合時間軸同步功能
- 支援大量資料點的效能優化

#### 2. 時間軸視覺化技術

**時間軸元件：**
- **Timeline.js：** 開源時間軸視覺化庫
- **Mermaid Timeline：** 適合文件中的時間軸展示
- **自訂時間軸元件：** 使用 React/Vue 建立客製化時間軸

**時間軸與地圖同步：**
- **狀態管理：** 使用 Redux/Vuex 管理時間軸與地圖狀態
- **即時更新：** 拖曳時間軸時即時更新地圖標記
- **效能優化：** 使用虛擬滾動處理大量時間點

#### 3. 資料處理與整合技術

**資料匯入與轉換：**
- **Google Sheets API：** 直接從 Google Sheets 匯入資料
- **CSV/JSON 處理：** 將試算表資料轉換為結構化 JSON
- **資料驗證：** 自動驗證資料完整性與格式

**地理編碼：**
- **OpenStreetMap Nominatim API：** 免費的地理編碼服務
- **Google Geocoding API：** 商業級地理編碼（需 API Key）
- **歷史地名對應：** 建立歷史地名與現代座標的對應表

**資料儲存：**
- **靜態 JSON 檔案：** 初期使用靜態檔案儲存資料
- **資料庫整合：** 後續可考慮 SQLite 或 PostgreSQL
- **資料快取：** 使用瀏覽器快取提升載入速度

#### 4. 前端框架與技術棧

**推薦技術棧：**
- **React + TypeScript：** 型別安全、元件化開發
- **Vue 3 + Composition API：** 輕量級、易於學習
- **狀態管理：** Redux Toolkit 或 Pinia

**地圖整合：**
- **React-Leaflet / Vue-Leaflet：** 框架專用的地圖元件
- **自訂地圖元件：** 封裝地圖功能為可重用元件

#### 5. 效能優化技術

**大量資料點處理：**
- **虛擬滾動：** 時間軸使用虛擬滾動處理大量時間點
- **標記聚合：** 地圖使用標記聚合（Marker Clustering）處理大量標記
- **懶加載：** 按需載入資料，減少初始載入時間

**互動效能：**
