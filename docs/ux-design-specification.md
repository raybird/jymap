**Author:** Kevin
**Date:** 2025-12-12

## Executive Summary

### Project Vision

**jymap** 是一個互動式金庸世界歷史地圖平台，整合時間軸、地圖視覺化與事件卡片，涵蓋全 15 部作品、跨朝代的完整時空脈絡。透過即時同步的時間軸與地圖互動，使用者可以深入理解金庸世界觀的時間與空間，並為未來的遊戲改編與教育應用奠定基礎。

**核心價值：** 將被動查詢轉變為主動探索體驗，讓金庸小說與歷史事件更生動地結合，提供更深層的理解與欣賞。

**產品定位：**
- 技術類型：Web Application（SPA，互動式地圖/時間軸）
- 領域：文化/歷史探索平台，兼具教育場景
- 複雜度：中等（核心挑戰在同步、效能、歷史地圖風格）

### Target Users

**核心用戶類型：**

1. **金庸群俠傳Online 玩家**
   - 目標：在遊戲遇到地點/事件時，快速查到真實地點與對應小說情節
   - 使用情境：遊戲中遇到地點，需要快速查詢對應的歷史背景與小說情節
   - 技術熟悉度：中等（熟悉遊戲介面，對地圖互動有一定理解）
   - 主要裝置：桌面（遊戲與查詢同時進行）

2. **電視劇/影集觀眾**
   - 目標：看完劇集後，理解劇情所處歷史脈絡與地理
   - 使用情境：觀看完劇集後，想要深入了解劇情背景、歷史事件與地理位置
   - 技術熟悉度：一般（熟悉基本網頁操作）
   - 主要裝置：桌面、平板（觀看後查詢）

3. **學生（歷史學習者）**
   - 目標：用地圖/時間軸建立時空脈絡，準備作業或自學
   - 使用情境：歷史學習、作業準備、自學探索
   - 技術熟悉度：一般至中等（熟悉網頁操作，可能使用過地圖工具）
   - 主要裝置：桌面、平板（學習場景）

**用戶共同需求：**
- 快速理解時空脈絡（時間與空間的對應關係）
- 探索相關事件（從單一事件擴展到多個相關事件）
- 驗證資料來源（確認資訊的可信度）
- 流暢的互動體驗（即時回應，無延遲感）

### Key Design Challenges

**1. 即時同步效能挑戰**
- **挑戰：** 時間軸與地圖同步需 <200ms，標記淡入淡出需流暢（60fps），同時支援 100+ 事件標記
- **影響：** 若同步延遲過高，會破壞「即時探索」的核心體驗
- **設計考量：** 需要優化渲染策略（標記聚合、漸進載入）、狀態管理（Redux Toolkit）、動畫效能（CSS transforms、GPU 加速）

**2. 視覺風格平衡挑戰**
- **挑戰：** Old Maps 風格底圖 + 中國水墨風格標記需保持可讀性，避免風格過重影響內容閱讀
- **影響：** 若視覺風格過重，可能影響標記與文字的清晰度，降低可用性
- **設計考量：** 需要平衡歷史感與可讀性，提供風格開關（標準/復古水墨），確保對比度符合 WCAG 2.1 AA 標準

**3. 複雜互動流程挑戰**
- **挑戰：** 時間軸拖曳、地圖互動、事件卡片展開、相關事件跳轉等多層互動需直覺且流暢
- **影響：** 若互動流程不直覺，使用者可能迷失在複雜的介面中
- **設計考量：** 需要清晰的視覺層次、一致的互動模式、明確的導覽路徑、鍵盤無障礙支援

**4. 資料密度管理挑戰**
- **挑戰：** 100+ 事件標記需流暢渲染，避免視覺混亂，同時保持探索的完整性
- **影響：** 若標記過於密集，可能造成視覺混亂，影響使用者找到目標事件
- **設計考量：** 需要標記聚合策略（Cluster）、視窗範圍優先載入、縮放層級控制、篩選機制（Post-MVP）

**5. 跨裝置體驗一致性挑戰**
- **挑戰：** 桌面優先設計需適配平板（iPad），保持拖曳與點擊精度
- **影響：** 若平板體驗不佳，可能影響部分用戶（學生、影集觀眾）的使用
- **設計考量：** 需要響應式設計、觸控手勢支援、平板專屬互動優化

### Design Opportunities

**1. 沉浸式探索體驗創新**
- **機會：** 從「被動查詢」轉為「主動探索」，透過時間軸拖曳與地圖互動創造探索感
- **設計方向：** 
  - 時間軸拖曳時的流暢動畫（標記淡入淡出、地圖平滑更新）
  - 事件卡片展開的過渡動畫（從標記到卡片的視覺連接）
  - 相關事件跳轉的視覺引導（動畫路徑、高亮提示）

**2. 視覺化創新與歷史感營造**
- **機會：** Old Maps 風格 + 中國水墨風格創造獨特的沉浸式體驗
- **設計方向：**
  - 歷史地圖底圖的復古質感（紙張紋理、褪色效果）
  - 標記與配色的水墨風格（毛筆筆觸、墨色漸變）
  - 事件卡片的歷史感設計（古書風格、印章元素）

**3. 資料透明化與可信度提升**
- **機會：** 每個事件附來源連結，提升資料可信度與可驗證性
- **設計方向：**
  - 來源連結的視覺呈現（圖示、顏色區分）
  - 來源缺失的明確標示（警告圖示、說明文字）
  - 資料驗證狀態的視覺反饋（已驗證/未驗證標記）

**4. 跨朝代連動探索體驗**
- **機會：** 支援跨朝代連續探索，讓使用者建立完整的時空脈絡認知
- **設計方向：**
  - 時間軸的朝代標記與視覺區分（顏色、標籤）
  - 跨朝代事件的視覺連接（時間線、關聯線）
  - 探索路徑的視覺化（使用者探索的歷史記錄）

**5. 效能優化與使用者體驗提升**
- **機會：** 透過效能優化（<200ms 同步、<500ms 卡片載入）創造流暢的互動體驗
- **設計方向：**
  - 載入狀態的視覺反饋（骨架屏、進度指示）
  - 互動回應的即時視覺反饋（按鈕狀態、動畫提示）
  - 效能監控與降級方案（標記聚合、漸進載入）

## Core User Experience

### Defining Experience
- 核心互動：搜尋 → 時間軸跳轉 → 地圖高亮標記 → 開啟事件卡 → 相關事件跳轉
- 主迴圈：滑動時間軸（或搜尋）即時驅動地圖；點標記展開卡片；從卡片再跳到相關事件，形成連續探索
- 必須順暢的關鍵：時間軸×地圖同步 <200ms、卡片開啟 <500ms、搜尋 <300ms（Top-3 ≥80%）

### Platform Strategy
- 平台：Web SPA；桌面優先，平板（iPad）支援；行動次要
- 操作方式：桌面以滑鼠/鍵盤；平板以觸控（拖曳、點擊、縮放）
- 瀏覽器：Chrome/Safari 最新兩版；無離線需求；無多人協作
- UI 重點：響應式布局、觸控手勢（時間軸拖曳、地圖縮放/平移）、鍵盤無障礙（搜尋/時間軸/標記）

### Effortless Interactions
- 搜尋即時：輸入即出結果（<300ms），點選後自動跳到時間軸位置並高亮地圖標記
- 同步無感：拖曳時間軸時，地圖標記淡入淡出，保持 <200ms 更新
- 卡片即達：點標記開卡片 <500ms，內容含歷史事件、小說情節、人物、來源；相關事件一鍵跳轉
- 導覽清晰：可見焦點、鍵盤導覽；平板觸控保持拖曳/點擊精度

### Critical Success Moments
- 首次搜尋→即時跳轉：使用者搜尋關鍵詞，看到時間軸定位與地圖高亮，同步無延遲
- 時間軸拖曳→標記更新：滑動時間軸時標記批次淡入淡出，無卡頓
- 點標記→卡片展開：卡片 <500ms 開啟，來源透明、資訊清晰
- 相關事件串連：從卡片跳轉到另一事件，地圖/時間軸保持上下文，不迷路
- 平板體驗：觸控拖曳與縮放順暢，點擊精度佳

### Experience Principles
- **主動探索優先**：時間軸×地圖同步為核心，鼓勵拖曳、點擊、跳轉的流動體驗
- **即時回饋**：搜尋、同步、卡片開啟皆需即時可感知的回應與動畫
- **可讀性優先於風格**：Old Maps + 水墨風格須保持對比與清晰度；必要時可切換標準風格
- **透明可信**：事件卡必附來源，來源缺失明示；資料可信度是體驗的一部分
- **低干擾、高導覽**：清楚的視覺層次與導覽（焦點狀態、狀態提示），避免過度裝飾干擾資訊辨識

## Desired Emotional Response

### Primary Emotional Goals
- **安心可信**：資料有來源、可驗證，避免質疑；看到來源與座標即建立信任
- **流暢掌控**：拖曳、縮放、搜尋、開卡片都不卡頓，保持掌控感與效率感
- **沉浸好奇**：Old Maps + 水墨風格帶來探索感，願意多跳轉、多發現
- **啟發與收穫**：在連續跳轉中獲得「原來如此」的理解（歷史/情節對應）

### Emotional Journey Mapping
- **初次進入**：好奇 + 安心（有清楚的導覽與來源透明提示）
- **核心探索（搜尋/拖曳/開卡片）**：流暢 + 掌控（即時同步、回饋明確）；無挫折
- **串連相關事件**：驚喜 + 啟發（發現跨朝代/跨作品的關聯）
- **完成後**：滿足 + 信任（覺得資訊可靠、探索有收穫）
- **再次回訪**：期待 + 熟悉（記得上次的流暢體驗，願意再來探索）

### Micro-Emotions
- **信任 vs. 質疑**：來源清楚、缺來源明示；地理座標可見，降低質疑
- **信心 vs. 迷失**：時間軸/地圖同步一致、相關事件跳轉保有上下文，不迷路
- **興奮 vs. 焦慮**：動畫流暢、載入快；避免等待與卡頓
- **滿足 vs. 挫折**：搜尋命中、卡片內容清晰；避免空結果或資訊不足

### Design Implications
- **建立信任**：事件卡固定展示來源區塊；缺來源明確標示；座標/朝代/作品清楚可見
- **流暢回饋**：搜尋/同步/卡片開啟皆有即時反饋與骨架屏；動畫連續不跳格
- **不迷路**：時間軸位置清楚、地圖高亮、相關事件跳轉保留上下文提示
- **減少焦慮**：保持 <200ms 同步、<300ms 搜尋、<500ms 卡片；顯示載入狀態
- **鼓勵探索**：相關事件區塊呈現清晰，提供快速跳轉與可視化路徑提示

### Emotional Design Principles
- **可信透明**：來源必顯、缺失必告；資訊可驗證是核心情感基礎
- **流暢掌控**：所有核心操作皆需即時可感的回饋與可預期的狀態
- **可導覽不迷路**：時間軸位置、地圖高亮、上下文提示，確保方向感
- **低摩擦高探索**：減少等待與步驟，保持探索連續性；相關事件引導好奇

## UX Pattern Analysis & Inspiration

### Inspiring Products Analysis

**1. Old Maps Online (https://www.oldmapsonline.org/)**

**核心問題解決：** 提供歷史地圖的數位化瀏覽與搜尋，讓使用者能夠探索不同時期的地圖樣貌。

**UX 成功要素：**
- **歷史地圖風格呈現：** 保留原始地圖的復古質感（紙張紋理、褪色效果、手繪風格），創造沉浸式歷史感
- **清晰的資訊層次：** 地圖為主要視覺，搜尋與篩選功能不干擾地圖瀏覽
- **流暢的縮放與平移：** 地圖互動流暢，支援多層級縮放（從世界到城市）
- **時間篩選機制：** 透過時間軸或年份選擇器過濾地圖，符合歷史探索需求

**最創新的互動：**
- 地圖與時間篩選的整合：選擇不同年份，地圖自動更新對應時期的地圖樣貌
- 地圖標記與資訊卡片的結合：點擊地圖標記，顯示地圖的詳細資訊（年代、來源、描述）

**視覺設計選擇：**
- 復古配色（米色、棕色、墨綠色）營造歷史感
- 地圖邊框與裝飾元素（印章、標題框）強化古地圖風格
- 清晰的對比度確保文字與標記可讀性

**錯誤處理：**
- 無結果時提供替代建議（相近年份、相近地點）
- 載入狀態明確（地圖載入進度、縮圖預覽）

**2. Google Maps (https://www.google.com/maps)**

**核心問題解決：** 提供全球地圖瀏覽、地點搜尋、路線規劃，讓使用者快速找到地點與導航。

**UX 成功要素：**
- **即時搜尋與自動完成：** 輸入即顯示搜尋建議（<300ms），Top-3 命中率高
- **流暢的地圖互動：** 縮放、平移、拖曳流暢（60fps），標記聚合（Cluster）處理大量標記
- **資訊卡片設計：** 點擊標記顯示地點卡片，包含名稱、地址、評分、照片、相關地點
- **視覺層次清晰：** 地圖為主要視覺，搜尋框、卡片、控制項不干擾地圖瀏覽

**最創新的互動：**
- 搜尋結果自動跳轉：點擊搜尋結果，地圖自動縮放並高亮對應標記
- 標記聚合：縮小地圖時，多個標記自動聚合為群組，點擊展開
- 相關地點推薦：地點卡片顯示「附近地點」、「相關地點」，支援快速跳轉

**視覺設計選擇：**
- 清晰的標記設計（顏色區分、圖示明確）
- 卡片設計簡潔（重要資訊優先、可展開詳細資訊）
- 動畫流暢（標記淡入淡出、卡片展開/收起）

**錯誤處理：**
- 搜尋無結果時提供替代建議（拼寫修正、相近地點）
- 網路錯誤時顯示離線提示與重試選項

**3. TimelineJS (https://timeline.knightlab.com/)**

**核心問題解決：** 提供時間軸視覺化工具，讓使用者能夠以時間線方式呈現歷史事件與故事。

**UX 成功要素：**
- **水平時間軸設計：** 時間軸水平滾動，事件標記清晰可見，支援拖曳與點擊
- **事件卡片整合：** 點擊時間軸標記，顯示事件卡片（標題、描述、圖片、來源）
- **流暢的動畫：** 時間軸滾動流暢，事件卡片淡入淡出，視覺連接清晰
- **響應式設計：** 桌面與平板適配良好，觸控手勢支援（滑動、點擊）

**最創新的互動：**
- 時間軸與事件卡片的同步：點擊時間軸標記，事件卡片自動展開並高亮
- 時間範圍選擇：支援框選時間範圍，過濾顯示該範圍內的事件
- 事件關聯視覺化：顯示事件之間的關聯線（時間線、因果鏈）

**視覺設計選擇：**
- 時間軸設計清晰（朝代標記、年份標籤、視覺區分）
- 事件卡片設計簡潔（標題、描述、圖片、來源連結）
- 配色與字體選擇符合歷史感（復古色調、清晰字體）

**錯誤處理：**
- 事件資料缺失時顯示預設內容（「資料載入中」、「來源缺失」）
- 時間軸載入狀態明確（進度條、骨架屏）

### Transferable UX Patterns

**導覽模式（Navigation Patterns）：**

1. **搜尋即時跳轉模式（來自 Google Maps）**
   - **適用場景：** jymap 的搜尋功能
   - **模式說明：** 輸入關鍵字即顯示搜尋建議（<300ms），點擊結果自動跳轉到對應位置（時間軸 + 地圖）
   - **設計要點：** 搜尋框置頂、結果列表清晰、點擊後同步更新時間軸與地圖

2. **時間軸水平滾動模式（來自 TimelineJS）**
   - **適用場景：** jymap 的時間軸設計
   - **模式說明：** 時間軸水平可拖曳，支援滑鼠拖曳與觸控滑動，事件標記清晰可見
   - **設計要點：** 時間軸置底或側邊、朝代標記與年份標籤、拖曳流暢（60fps）

3. **地圖標記聚合模式（來自 Google Maps）**
   - **適用場景：** jymap 的 100+ 事件標記渲染
   - **模式說明：** 縮小地圖時，多個標記自動聚合為群組（Cluster），點擊展開
   - **設計要點：** 聚合標記顯示數量、點擊展開動畫、視窗範圍優先載入

**互動模式（Interaction Patterns）：**

1. **標記點擊→卡片展開模式（來自 Google Maps + TimelineJS）**
   - **適用場景：** jymap 的事件卡片功能
   - **模式說明：** 點擊地圖標記或時間軸標記，事件卡片自動展開（<500ms），包含詳細資訊
   - **設計要點：** 卡片位置不遮擋標記、展開動畫流暢、內容層次清晰（歷史事件、小說情節、人物、來源）

2. **時間軸×地圖同步模式（來自 Old Maps Online + TimelineJS）**
   - **適用場景：** jymap 的核心互動
   - **模式說明：** 拖曳時間軸時，地圖標記自動更新（<200ms），標記淡入淡出
   - **設計要點：** 同步延遲 <200ms、標記動畫流暢（60fps）、狀態管理清晰（Redux Toolkit）

3. **相關事件跳轉模式（來自 Google Maps 相關地點推薦）**
   - **適用場景：** jymap 的相關事件功能
   - **模式說明：** 事件卡片顯示「相關事件」區塊，點擊後自動跳轉到對應位置（時間軸 + 地圖）
   - **設計要點：** 相關事件列表清晰、跳轉動畫流暢、上下文提示明確

**視覺模式（Visual Patterns）：**

1. **歷史地圖風格底圖（來自 Old Maps Online）**
   - **適用場景：** jymap 的地圖底圖設計
   - **模式說明：** 使用復古地圖風格（紙張紋理、褪色效果、手繪風格），略帶中國水墨風格
   - **設計要點：** 保持可讀性（對比度 WCAG AA）、風格開關（標準/復古水墨）、標記與文字清晰

2. **事件卡片歷史感設計（來自 TimelineJS + Old Maps Online）**
   - **適用場景：** jymap 的事件卡片設計
   - **模式說明：** 卡片設計融入歷史感元素（古書風格、印章元素、復古配色）
   - **設計要點：** 內容優先（資訊清晰）、風格適度（不干擾閱讀）、來源透明（必附來源連結）

3. **時間軸朝代視覺區分（來自 TimelineJS）**
   - **適用場景：** jymap 的時間軸設計
   - **模式說明：** 時間軸以顏色、標籤區分不同朝代（春秋、北宋、南宋、元、明、清）
   - **設計要點：** 朝代標記清晰、顏色對比明顯、年份標籤可見

### Anti-Patterns to Avoid

**1. 過度裝飾影響可讀性**
- **問題：** 歷史地圖風格過重，影響標記與文字的清晰度
- **避免方式：** 保持對比度（WCAG AA）、提供風格開關、優先可讀性

**2. 同步延遲破壞體驗**
- **問題：** 時間軸與地圖同步延遲 >200ms，造成卡頓感
- **避免方式：** 優化渲染策略（標記聚合、漸進載入）、狀態管理（Redux Toolkit）、動畫效能（CSS transforms、GPU 加速）

**3. 資訊卡片遮擋地圖**
- **問題：** 事件卡片展開時遮擋地圖標記，影響探索
- **避免方式：** 卡片位置優化（側邊、底部、浮動）、可收起、不遮擋關鍵標記

**4. 搜尋結果不準確**
- **問題：** 搜尋 Top-3 命中率 <80%，使用者找不到目標事件
- **避免方式：** 使用全文搜尋庫（Fuse.js）、建立搜尋索引、A/B 測試搜尋演算法

**5. 標記過於密集造成視覺混亂**
- **問題：** 100+ 標記同時顯示，造成視覺混亂，難以找到目標事件
- **避免方式：** 標記聚合（Cluster）、視窗範圍優先載入、縮放層級控制、篩選機制（Post-MVP）

**6. 相關事件跳轉迷失方向**
- **問題：** 從事件卡片跳轉到相關事件時，使用者迷失在複雜的介面中
- **避免方式：** 上下文提示明確（時間軸位置、地圖高亮）、探索路徑視覺化、返回機制

## Design System Foundation

### 1.1 Design System Choice
- **主選**：MUI Core v5（MIT 授權，僅使用免費套件）+ 自訂 Theme/Design Tokens；必要時少量搭配 Headless UI/Tailwind 用於特殊浮層/彈窗或動畫細節。
- **明確排除**：不使用 MUI Pro/Premium（避免授權與收費疑慮）。
- **備援**：若後期需更高視覺差異化，保持同一組 tokens，漸進覆寫 MUI 核心元件樣式（不更換技術棧）。

### Rationale for Selection
- **授權清晰**：MUI Core 為 MIT，無付費套件依賴；符合授權合規與成本控管。
- **速度與覆蓋度（PM + Eng）**：元件完整、文檔成熟，桌面優先 + 平板快速交付；React/TS 相容，與既定技術棧一致。
- **可讀性與風格彈性（Design）**：Theme 易於套用 Old Maps + 水墨色盤，並可提供標準/復古水墨雙主題，保證可讀性優先（WCAG AA）。
- **效能與維護（Eng）**：Tree-shaking、按需載入；大社群、維護成本可控；可與 Leaflet/Mapbox 分層協作，避免佈局耦合。
- **無障礙基礎（Design + Eng）**：MUI 內建 ARIA/a11y；可微調焦點態與可見度，符合搜尋/時間軸/標記的鍵盤導覽要求。

### Implementation Approach
- **Theme/Tokens**：色票（底圖復古棕/米色、墨黑高亮、朱紅點綴）、字體（可讀中文+英文）、間距、陰影、動效曲線。
- **元件策略**：MUI 核心元件（Button, Tabs, Drawer/Sheet, Dialog, Card, List, Tooltip）；時間軸/地圖控制為客製組件；必要時用 Headless/Tailwind 做特殊浮層。
- **佈局**：響應式斷點（桌面優先，平板適配）；頂部搜尋列 + 底部/側邊時間軸 + 主視圖地圖。
- **效能**：按需載入、虛擬滾動（列表/搜尋結果）、CSS transforms 動效；Leaflet/Mapbox 自行渲染，不與 MUI 佈局耦合。
- **無障礙**：保留並微調焦點態；ARIA label/description 套用於搜尋、時間軸控制、標記列表。

### Customization Strategy
- **色彩/樣式**：雙主題（標準 / 復古水墨）；保持對比度達 WCAG AA；必要時提供風格開關。
- **字體/圖標**：中文可讀性字體 + 輕量圖標；標記/卡片圖示統一風格。
- **元件覆寫**：Card（歷史感邊框/標題）、Tabs（時間軸切換）、Drawer/Sheet（事件卡/相關事件）、Tooltip（標記提示）。
- **客製組件**：時間軸（水平拖曳、朝代顏色帶）、地圖浮層控制（縮放、聚合圖例）、相關事件清單。
- **演進路徑**：若需更高差異化，保持 tokens 不變，逐步覆寫 MUI 樣式；避免改動基礎架構以控管風險。

## Visual Design Foundation

### Color System
- **雙主題**：標準 / 復古水墨（可切換，優先可讀性）
- **主色盤（復古水墨）**：背景米色/淡棕（Old Maps 紙質感）、墨黑（文字/高亮）、朱紅（點綴/強調）、墨綠/靛藍（次要高亮）、淺灰（分隔/底）
- **主色盤（標準）**：中性色淺底 + 深字，保留朱紅/墨綠作為點綴，提升對比
- **語意色**：Primary=朱紅、Secondary=墨綠/靛藍、Success=墨綠、Warning=赭黃、Error=深紅；確保 WCAG AA 對比
- **可讀性**：對比度達 WCAG 2.1 AA；標記與文字在 Old Maps 底圖上仍清晰；提供風格開關避免過度裝飾

### Typography System
- **語調**：沉浸、可信、易讀；歷史感但不花俏
- **字體**：中文選用易讀黑體/宋體系（如 Noto Sans TC / Noto Serif TC 視實際可用性），英文字體搭配無襯線（如 Inter/Roboto）以確保對比
- **層級**：標題（h1-h3）清晰對比，正文字級偏大以便閱讀卡片；行高 1.5-1.6；卡片與列表保持足夠行距
- **用法**：卡片標題強調、正文簡潔；時間軸刻度與朝代標籤需清晰可辨；搜尋結果列表字級不宜過小

### Spacing & Layout Foundation
- **Spacing 單位**：8px 基準，必要時 4px 細調；主要間距使用 8/16/24/32 階梯
- **密度**：中等密度，地圖為主視覺，介面控件保持呼吸感以避免擋住地圖
- **版面**：桌面優先，平板適配；頂部搜尋、底部/側邊時間軸、主視圖為地圖；卡片側邊或底部抽屜，避免遮擋標記
- **網格**：12 欄（桌面）；平板可 8 欄；主要視覺集中於地圖區，輔以固定搜尋列與可收合側欄/抽屜

### Accessibility Considerations
- **對比與字級**：對比度達 WCAG AA；正文與標籤字級不小於無障礙建議；行高充足
- **鍵盤操作**：搜尋、時間軸控制、標記列表與卡片皆可鍵盤導覽；焦點可見
- **ARIA**：搜尋框、時間軸控制、標記/卡片元素具 aria-label/description；動態更新（標記刷新、搜尋結果）以 ARIA live region 提示
- **風格開關**：提供標準/復古水墨切換；在復古風格下仍維持可讀性與對比，避免過度紋理干擾

## Design Direction Decision

### Design Directions Explored
- **Direction A — Standard Clean**：中性色、淺底 + 深字，重視可讀性與效能，Old Maps 元素僅作點綴。
- **Direction B — Classic Old Maps**：復古紙感底圖＋水墨色盤，強化沉浸感，標記/文字對比經過加強。
- **Direction C — Hybrid (Clean + Old Maps Accents)**：以標準清爽布局為主，局部套用復古紋理與水墨配色（底圖輕紋理、標記與卡片用墨色/朱紅點綴），保證可讀性與效能。

### Chosen Direction
- **Direction C — Hybrid (Clean + Old Maps Accents)**  
  - 主體走乾淨、清晰的可讀性；以雙主題切換（標準 / 復古水墨），預設採 Hybrid：淺底、清楚對比 + 局部復古紋理與墨色點綴。

### Design Rationale
- **可讀性與效能優先**：核心互動要求同步 <200ms、搜尋 <300ms、卡片 <500ms，需清爽介面減少視覺噪音。
- **沉浸與品牌感**：保留 Old Maps + 水墨元素於底圖紋理、標記配色、卡片裝飾，營造歷史感但不影響清晰度。
- **可切換風格**：標準/復古水墨雙主題，因應可讀性與偏好；滿足「略帶中國水墨風格」且可驗證資料透明。
- **跨裝置一致**：桌面優先、平板適配，Hybrid 佈局在兩端保持清楚導覽與焦點狀態。

### Implementation Approach
- **色彩應用**：預設淺底 + 墨黑文字 + 朱紅點綴；復古主題時啟用輕紋理底圖，確保對比達 WCAG AA。
- **元件套用**：MUI Core + 自訂 tokens；卡片/標記/時間軸套用墨色/朱紅；重要 CTA 使用高對比色。
- **佈局**：頂部搜尋、底部/側邊時間軸、主視圖地圖；卡片側邊/底部抽屜不遮標記；標記聚合提示清晰。
- **無障礙**：焦點可見、ARIA 描述、live region 提示；風格切換不降低對比。

## User Journey Flows

### Journey 1：玩家（搜尋 + 時間軸跳轉）

描述：遊玩中遇到地點/事件時，快速查到真實地點與小說情節，並探索相關事件。

```mermaid
flowchart TD
  A[搜尋輸入關鍵詞] --> B{即時搜尋結果<br/>(<300ms)}
  B -->|點擊結果| C[時間軸自動跳轉/高亮]
  C --> D[地圖高亮對應標記]
  D --> E[點標記開事件卡<br/>(<500ms)]
  E --> F{相關事件列表}
  F -->|點擊相關事件| C
  E --> G[查看來源/座標/朝代]
  B -->|無結果| H[提供建議/拼字修正]
```

重點節點：搜尋即時回饋；跳轉同步 <200ms；卡片不遮地圖；相關事件跳轉保留上下文。

### Journey 2：影集觀眾（時間軸導覽 + 閱讀卡片）

描述：看完劇集後，沿時間軸拖曳瀏覽連續劇情，點標記閱讀卡片並跳轉相關事件。

```mermaid
flowchart TD
  A[預設停在劇集年代] --> B[拖曳時間軸]
  B --> C[標記批次淡入淡出<br/>(<200ms)]
  C --> D[點標記開卡片<br/>(<500ms)]
  D --> E[展開相關事件並跳轉]
  D --> F[來源/情節摘要/人物]
  B --> G{區間選擇?}
  G -->|是| H[只顯示該區間標記]
```

重點節點：時間軸拖曳流暢；標記淡入淡出；卡片內容清晰且可展開相關事件；區間範圍過濾。

### Journey 3：學生（歷史學習，區間篩選 + 串連）

描述：搜尋歷史事件/戰役，框選時間區間，串連多事件建立時空脈絡。

```mermaid
flowchart TD
  A[搜尋「靖康」「蒙古入侵」] --> B{搜尋結果列表}
  B -->|選擇事件| C[時間軸跳轉/地圖高亮]
  C --> D[框選時間區間 1200-1280]
  D --> E[地圖僅顯示該段標記]
  E --> F[點標記開卡片]
  F --> G[相關事件跳轉<br/>(地點/人物關聯)]
  B -->|無結果| H[提示更廣關鍵字/相近事件]
```

重點節點：區間篩選；時間軸/地圖同步過濾；相關事件基於地點/人物串連；缺結果時的替代建議。

### Journey Patterns
- 導覽：搜尋即時建議 + 自動跳轉時間軸/地圖；時間軸水平拖曳/區間選擇；卡片側/底抽屜不遮標記。
- 回饋：同步 <200ms、搜尋 <300ms、卡片 <500ms；骨架屏/高亮/淡入淡出；上下文提示（時間軸位置、朝代標記、地圖高亮）。
- 決策：相關事件清單清楚；無結果時提供建議/修正；跳轉後保留返回/路徑線索。

### Flow Optimization Principles
- 減步驟：搜尋點擊即跳轉並高亮；卡片一鍵展開相關事件。
- 降負荷：標記聚合避免雜訊；區間篩選聚焦當期；卡片資訊分區（事件/情節/人物/來源）。
- 保流暢：同步/搜尋/卡片性能指標達標；動畫連續不跳格。
- 防迷路：上下文提示與返回路徑；相關跳轉仍顯示當前時間與地理焦點。

## Component Strategy

### Design System Components
- **採用**：MUI Core v5（MIT，免費套件），自訂 tokens；必要時少量 Headless UI/Tailwind 處理特殊浮層/動畫。
- **現成可用**（MUI）：Button、Tabs、Drawer/Sheet、Dialog、Card、List、Tooltip、Menu、Popover、Breadcrumb、Stepper、Pagination、Accordion。
- **覆寫重點**：Card（歷史感邊框/標題）、Tabs（時間軸切換）、Drawer/Sheet（事件卡/相關事件）、Tooltip（標記提示）、Menu/Popover（搜尋建議、相關事件）。

### Custom Components
- **時間軸組件（水平拖曳 + 朝代視覺帶）**  
  - Purpose：主迴圈核心，拖曳/跳轉、顯示朝代分段與事件標記。  
  - States：預設/拖曳/高亮當前時段；區間選擇；無資料提示。  
  - Variants：底部（桌面）、側邊/底部（平板）；標準/復古水墨主題。  
  - Accessibility：鍵盤左右/上下跳轉、Tab 聚焦標記；ARIA 描述當前年份/朝代。  

- **地圖標記 + 聚合（Cluster）**  
  - Purpose：在 100+ 事件下仍保持清晰；縮放聚合、點擊展開。  
  - States：預設/高亮/聚合/選取；載入中。  
  - Variants：標準/復古水墨配色；大/中標記尺寸。  
  - Accessibility：鍵盤選取、焦點高亮；ARIA label（地點/事件/年份）。  

- **事件卡片（側邊/底部抽屜）**  
  - Purpose：顯示歷史事件、小說情節、人物、來源、相關事件跳轉。  
  - Content：標題、朝代/年份、地點、小說對應、人物、來源連結、相關事件列表。  
  - States：預設/展開/載入骨架/無來源提示。  
  - Variants：側邊抽屜（桌面）、底部抽屜（平板）；標準/復古水墨主題。  
  - Accessibility：Escape 關閉、Tab 巡覽、焦點可見；ARIA 標記來源/相關事件列表。  

- **搜尋結果下拉/建議清單**  
  - Purpose：即時搜尋（<300ms），列出 Top-3+ 建議，點擊自動跳轉時間軸+地圖。  
  - States：預設/載入/無結果/錯誤提示；選取高亮。  
  - Variants：標準/復古水墨；支援鍵盤上下選取、Enter 確認。  
  - Accessibility：ARIA live region 報讀結果數；aria-activedescendant 標記高亮項。  

- **相關事件列表（卡片內）**  
  - Purpose：跨朝代/人物/地點的快速跳轉。  
  - States：預設/載入/無相關；點擊即跳轉時間軸+地圖。  
  - Variants：列表或小卡；標準/復古水墨。  
  - Accessibility：鍵盤可跳轉、aria-label 說明目標事件與年代。  

- **區間篩選控件（時間軸區間框選）**  
  - Purpose：學生/研究場景，框選時間範圍進行過濾。  
  - States：預設/選取中/調整中；無資料提示。  
  - Variants：拖曳柄、數字輸入（年份）；標準/復古水墨。  
  - Accessibility：鍵盤調整範圍，ARIA 描述當前範圍。  

### Component Implementation Strategy
- **基礎元件**：優先使用 MUI Core，透過 tokens 覆寫樣式（色彩、圓角、陰影、字體）。  
- **客製元件**：時間軸、地圖標記/聚合、事件卡、相關事件列表、區間篩選。  
- **一致性**：採用雙主題 tokens（標準/復古水墨）；保持對比度 AA；動畫用 CSS transforms，避免 jank。  
- **無障礙**：ARIA label/description、焦點樣式、鍵盤操作；動態更新使用 live regions。  
- **效能**：按需載入、虛擬滾動（列表/建議）、標記聚合、漸進載入，保持同步 <200ms、搜尋 <300ms、卡片 <500ms。  

### Implementation Roadmap
- **Phase 1（MVP 核心）**：搜尋下拉/建議、時間軸（拖曳/跳轉/朝代帶）、地圖標記+聚合、事件卡片（來源/相關事件）、相關事件列表、區間篩選（基礎）。
- **Phase 2（成長）**：標記聚合優化（500+）、進階相關事件視覺化、時間軸區間交互優化、列表虛擬滾動強化。
- **Phase 3（擴展）**：個人化/收藏、進階視覺化（勢力/戰役路徑）、教育/遊戲化附加元件。

## UX Consistency Patterns

### Button Hierarchy
- Primary：高對比色（標準：深/亮紅；復古：朱紅）用於核心行動（搜尋觸發、跳轉確認）。
- Secondary：中性/墨綠，用於次要操作（切換、篩選）。
- Tertiary/Text：中性色文字按鈕無邊框，用於輔助（返回、更多）；Icon Button 只用於明確圖示（返回/關閉/定位），必備 aria-label。
- States：hover/active 對比明顯；disabled 降低不透明度；Focus 有明顯描邊/陰影。

### Feedback Patterns
- 成功：綠色訊息條/Toast，短訊息；可出現在右上或卡片內。
- 錯誤：紅色訊息條/行內提示；欄位錯誤附說明與圖示；提供重試/修正指引。
- 載入：骨架屏（卡片/列表）、標記淡佔位；搜尋/同步顯示 spinner 或細條進度。
- 空狀態：清晰圖示 + 簡訊 + 行動（重新搜尋/放寬範圍）。
- 實時更新：搜尋結果/標記更新顯示短提示「已更新」；使用 ARIA live region。

### Form Patterns（搜尋/篩選）
- 搜尋：置頂單欄，debounce + 即時建議，鍵盤上下/Enter；無結果給建議/拼字修正。
- 篩選（區間/條件）：滑桿或年份輸入，預設區間，可重設；套用時顯示載入狀態。
- 驗證：錯誤訊息靠近欄位且簡短。
- 可及性：aria-label/description，Tab 順序合邏輯；Enter 提交、Esc 關閉。

### Navigation Patterns
- 全局：頂部搜尋 + 主視圖地圖 + 底/側時間軸；卡片抽屜不遮關鍵標記。
- 相關跳轉：卡片內列表，點擊即同步時間軸+地圖；保留返回/路徑提示。
- 區間視圖：時間軸框選顯示當前範圍標籤；可重設。
- 列表：優先虛擬滾動；若分頁，顯示總數與當前範圍。

### Additional Patterns
- Modal/Overlay：少用；優先 Drawer/Sheet，避免遮地圖。
- Tooltip/Popover：標記提示、搜尋建議、相關事件短訊息；Esc/點外可關閉，對比清晰。
- 搜尋/同步錯誤：友善提示 + 重試；網路錯誤顯示重試按鈕。
- Loading/Empty for Map：標記骨架/淡佔位 + 短訊息；空結果提示放寬條件。
- Accessibility：焦點樣式一致；ARIA live region 用於搜尋結果、標記更新；風格切換不犧牲對比。

## Responsive Design & Accessibility

### Responsive Strategy
- **Desktop**：桌面優先；12 欄格，頂部搜尋、底/側時間軸、主視圖地圖；卡片側邊抽屜（不遮標記）。利用較大視窗同時展示搜尋結果/卡片與地圖。
- **Tablet（iPad）**：8 欄格，觸控優化；時間軸可置底或側；卡片改底部抽屜，手勢拖曳/點擊精度優先；控件增大觸控區域。
- **Mobile（次要）**：若需支援，簡化為底部導航/漢堡；時間軸折疊為縮略帶，卡片全屏疊加；優先搜尋與單一重點標記展示。

### Breakpoint Strategy
- 建議使用標準斷點：Mobile <768px、Tablet 768-1023px、Desktop ≥1024px；桌面優先思維，但媒體查詢採 mobile-first 寫法。
- 平板重點：觸控區域 ≥44px，手勢拖曳（時間軸/地圖），抽屜高度可調。

### Accessibility Strategy
- 目標 WCAG 2.1 AA：對比度 4.5:1（文字）、3:1（UI 元件）；字級/行高遵循 AA。
- 鍵盤：搜尋/時間軸/標記/卡片全鍵盤可用，焦點可見，Esc 關閉抽屜/浮層。
- ARIA：搜尋（aria-label/描述 + live region 報讀結果）、時間軸控制（當前年份/朝代描述）、標記/卡片（地點/事件/年份/來源）、相關事件列表（目標事件與年代）。
- 觸控尺寸：主要可點元素 ≥44x44px；間距保持 8px 基準並確保觸控可用。
- 風格開關：標準/復古水墨主題切換不降低對比與可讀性。

### Testing Strategy
- 響應式：實機/瀏覽器（Chrome/Safari/Firefox/Edge）測試；桌面+平板為主；網路性能（3G/4G）下的載入體驗。
- 無障礙：自動化 a11y 掃描（lint/axe）；螢幕閱讀器（VoiceOver, NVDA）檢測；鍵盤巡覽；色弱模擬。
- 使用者測試：含觸控場景；蒐集搜尋/同步/卡片速度體感。

### Implementation Guidelines
- RWD：mobile-first media queries；使用 rem/%/vw/vh；圖片/切片優化；觸控手勢與滑鼠並行。
- a11y：語意化 HTML；ARIA label/role；焦點管理與 skip links；保持高對比模式；動態更新用 live region；表單錯誤行內提示。

## 2. Core User Experience

### 2.1 Defining Experience
- **一句話核心互動**：搜尋或拖曳時間軸 → 地圖即時高亮對應事件 → 點標記開卡片 → 相關事件一鍵跳轉，形成連續探索。
- **使用者向朋友描述時**：「我拖時間軸，地圖立刻亮出該時期的江湖/歷史事件，點一下就看到卡片與來源，還能跳到相關事件一路逛下去。」

### 2.2 User Mental Model
- **現有心智模型**：像地圖搜尋（Google Maps）、時間線導覽（影集進度條/TimelineJS），期待「拖動即更新」且「點即看卡片」。
- **預期行為**：搜尋即出結果並自動定位；拖曳時標記同步淡入淡出；點標記開卡片不遮擋地圖，資訊分明。
- **可能困惑**：跨朝代跳轉時的上下文；標記過多時的視覺混亂；卡片開啟位置與返回路徑。

### 2.3 Success Criteria
- **體感速度**：同步 <200ms；搜尋 <300ms；卡片 <500ms；動畫流暢（60fps）。
- **即時理解**：時間軸位置與朝代顯示清楚；地圖高亮對應時期/事件；卡片首屏呈現關鍵資訊+來源。
- **不中斷探索**：相關事件跳轉後，仍保有上下文提示（時間軸位置、地圖高亮、返回/路徑線索）。
- **可信透明**：每張卡片均有來源；缺來源明示；座標/朝代/作品資訊清楚。

### 2.4 Novel UX Patterns
- **熟悉套用**：沿用已知模式（Maps 搜尋→定位、高亮；Timeline 水平拖曳；標記→卡片）。
- **創新融合**：時間軸×地圖雙向同步 + 相關事件的跨朝代跳轉。需在卡片與時間軸上提供上下文提示，避免迷路。
- **引導策略**：首次使用時，顯示簡短導覽（拖時間軸、點標記、卡片內的相關事件跳轉）；提供風格開關避免過度裝飾影響可讀性。

### 2.5 Experience Mechanics
- **Initiation**：搜尋框輸入或直接拖曳時間軸；預設停在熱門/預設時期，顯示核心標記。
- **Interaction**：拖曳時間軸 → 標記批次淡入淡出同步；點標記 → 卡片展開（側邊/底部，不遮擋）；卡片內點「相關事件」→ 時間軸與地圖同步跳轉。
- **Feedback**：載入骨架屏/進度；時間軸位置與朝代顯示；標記高亮；卡片展開動畫；搜尋/同步/卡片的即時回饋。
- **Completion**：使用者成功查看一張卡片並跳轉至少一個相關事件，且未迷路（仍能辨識當前時間位置與地理焦點）；可隨時關閉卡片返回地圖。
