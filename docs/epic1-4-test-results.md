### ❌ 發現的問題

#### 問題 1: 地圖標記沒有顯示（嚴重）✅ 已修復

**問題描述：**
- 地圖上應該顯示 3 個事件標記，但實際顯示 0 個標記
- 控制台沒有「已在地圖上添加 X 個標記」的訊息

**影響範圍：**
- Epic 2: 地圖標記渲染（Story 2.3）
- Epic 4: 時間軸與地圖同步（Story 4.2）
- FR6: 使用者可以透過地圖縮放與平移探索不同地理區域
- FR7: 使用者可以透過點擊地圖標記查看該地點的詳細事件資訊

**根本原因：**
1. 訂閱條件錯誤：`events.length >= 0` 永遠為 true，應該改為 `events.length > 0`
2. 時機問題：訂閱在 constructor 中建立，但地圖在 `ngAfterViewInit` 中初始化。如果事件在訂閱建立時已經載入完成，但地圖還沒初始化，就會錯過更新

**修復方案：**
1. ✅ 修正訂閱條件：`events.length > 0`
2. ✅ 在地圖初始化完成後，使用 `take(1)` 獲取當前事件並添加標記
3. ✅ 確保在地圖初始化完成後，如果已有事件資料，立即添加標記

**修復結果：**
- ✅ 地圖上成功顯示 3 個標記
- ✅ 控制台顯示「已更新地圖標記：移除 0 個，添加 3 個」

#### 問題 2: 時間軸年份刻度可見性 ✅ 已改善

**問題描述：**
- 時間軸只顯示朝代帶，年份刻度可能因為對比度不足而不明顯
- 年份標籤的可讀性需要改善

**影響範圍：**
- Epic 3: 時間軸視覺化（Story 3.2）
- FR1: 使用者可以透過水平拖曳時間軸探索不同歷史時期

**修復方案：**
1. ✅ 增加年份刻度的對比度（從 `rgba(0, 0, 0, 0.2)` 改為 `rgba(0, 0, 0, 0.3)`）
2. ✅ 改善年份標籤的可讀性：
   - 增加字體顏色對比度（從 `0.7` 改為 `0.8`）
   - 增加字重（從 `500` 改為 `600`）
   - 增強文字陰影
   - 添加半透明背景以提高可讀性
3. ✅ 確保年份刻度不阻擋拖曳操作（`pointer-events: none`）

**修復結果：**
- ✅ 年份刻度對比度提升，更容易看見
- ✅ 年份標籤可讀性改善
